PlateCalculator algorithm — greedy heaviest-first (RESEARCH §4 cites tommyodland.com canonical-coin-system proof):
1. `weightPerSide = (target - barWeight) / 2.0`, rounded to 3 decimal places to dodge float drift.
2. Sort plates descending by weight.
3. For each plate: `useCount = min(plate.countPerSide, Int(remaining / plate.weight))`. Subtract; re-round to 3 places.
4. Compare remaining with epsilon 0.001 (not == 0).
5. Return PlateStack {platesPerSide, totalWeight = barWeight + 2*sumPerSide} on success; nil on failure (solve()) or barWeight (roundDown for target<bar).

Calibration weighted-mean algorithm (RESEARCH §3 simplified — degenerate LOWESS at polynomial degree 0):
- Gaussian kernel: `w_i = exp(-(daysFromNow_i / 30.0)²)` where daysFromNow_i = (now - point.date) in days.
- `calibratedE1RM = sum(w_i * e1RM_i) / sum(w_i)` — guard `sum(w_i) >= 1e-9` else return nil.
- Empty history → nil. The caller (RPEAutoregStrategy in plan 03-05) is responsible for filtering nil-RPE history before calling.
