<objective>
Ship the **Slice E vertical**: a serious lifter who has been pushing hard for 3 weeks opens the Today tab on a Monday morning and sees an amber banner at the top of the screen reading "Consider deload — e1RM dropped 6% on chest lifts over last 3 sessions. Tap to see signals." The banner is dismissible (xmark) and tapping its body opens a detail sheet listing each firing signal with numeric values. Volume bars on the fatigue surface carry an amber ring tint while the advisory is active.

This plan REPLACES Phase 4's `StubFatigueAdvisory` (which returns false) with the real `DeloadAdvisor` implementation — the type-level canonicality enforcement (BLOCK-08 + T-05-03) is preserved: DeloadAdvisor returns advisory data (a value type that the compiler refuses to coerce into a `DeloadMutation`), never a schedule mutation. The only writer of block deload state remains `PeriodizationEngine.advance(...)` per Phase 4 D-25.

Purpose: this is the system's "I noticed something wrong" signal — distinct from the per-exercise plateau detector (plan 03), which fires when ONE exercise stalls. The deload advisory fires when the user's whole-body training stress shows fatigue across multiple exercises and signals. The single-banner top-of-Today-tab placement is the planner's bet on attention economy: the user sees the warning once per Monday they don't dismiss, and the detail sheet provides transparency without nagging (no notifications, per D-16).

Output: 1 new pure-function service (`DeloadAdvisor`), 2 new SwiftUI views (`DeloadAdvisoryBanner`, `DeloadSignalDetailSheet`), 1 modified host (`RootView`), 1 modified view (`MuscleVolumeBar` — amber ring overlay), 1 GREEN unit test suite (DeloadAdvisor; replacing 1 of the 8 RED scaffolds from plan 01), 1 GREEN UI test suite (DeloadAdvisoryBanner).

End-to-end user-visible deliverable: simulating an e1RM drop across pooled top-sets fires the advisory; the amber banner appears at the top of Today tab with the verbatim copy template; tapping opens the signal-detail sheet (medium detent); tapping dismiss hides the banner for the current week (xmark sets settings.deloadAdvisoryDismissedWeekStart = currentMonday); re-evaluating on next Monday surfaces the banner again if signals persist. Volume bars carry amber ring tint while the advisory is active.
</objective>

    3. Compute Sig-2 — group sessions by (exercise.id, intentRaw) pair. Compute `missRatio = missed.count / allWorkingSets.count`. If missRatio > 0.5, sig3 fires.
