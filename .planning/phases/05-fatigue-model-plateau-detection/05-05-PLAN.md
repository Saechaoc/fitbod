<objective>
Ship the **Slice C vertical**: a serious lifter opens the Today tab, scrolls past the Volume This Week bars, and sees a Muscle Heatmap — a body silhouette (front view by default, toggleable to back) with each muscle region tinted in its zone color (4-zone palette matching the bars). Tapping the chest region — or tapping the chest bar — pushes onto the navigation stack a PerMuscleDetailView for the chest muscle showing: (1) This Week section (set count, zone badge with verb abbreviation, delta), (2) Contributing Exercises section (sorted DESC by weighted-set count this week), (3) Frequency This Week section (count of sessions hitting ≥ frequencyHitMinSets), and (4) Adjust Targets section with an inline MuscleVolumeTargetStepper (4 Steppers + Reset button — closes the SET-05 inline entry point).

This plan refines plan 01's stub `MuscleRegionPaths` rectangles to anatomically-correct bezier curves per RESEARCH §8.3 hand-coded pattern. Visual fidelity is the planner's bet on "the heatmap is the *visual* way to feel your weekly volume" — bars give the user numbers, but the heatmap gives them a body-shaped intuition of where they trained.

Purpose: VOL-05 closure (front + back silhouette + drillable per-muscle detail + SET-05 inline editor entry point). VOL-06 gets a UI surface (Frequency This Week section in PerMuscleDetailView consumes plan 02's FatigueModel.frequencyHits) even though the service was service-side closed in plan 02.

Output: 1 substantially-modified file (`MuscleRegionPaths.swift` — adding ~17 × 2 ≈ 34 anatomically-correct paths replacing stub rects), 3 new SwiftUI views (`BodySilhouetteView`, `PerMuscleDetailView`, `MuscleVolumeTargetStepper`), 2 modified Phase 5 views (`FatigueSurfaceView` for heatmap mount + navigation; `MuscleVolumeBar` for onTap wire-up), 2 GREEN UI test suites (BodySilhouetteUITests + PerMuscleDetailUITests).

End-to-end user-visible deliverable: Today tab shows the Muscle Heatmap section with both front and back silhouettes (toggle via segmented control) — chest region is tinted per the current week's chest zone; tapping it opens PerMuscleDetailView with title "Chest" and all 4 sections populated; tapping "Adjust Targets" reveals the inline editor; the user edits MEV from 8 to 10 and the bar zone-recompute reflects the new MEV immediately (round-trip via SwiftData @Bindable).

        @MainActor
        private func computeContributingExercises() -> [ContributingExercise] {
            // Fetch all working SetEntry rows in the week window for this muscle's stimulus.
            // In-memory aggregate by Exercise.canonicalName.
            // Return sorted DESC by total weighted sets.
            let weekEnd = weekStart.addingTimeInterval(7 * 86400)
            let workingKinds = ["working", "drop", "failure", "rest_pause"]
            let captureStart = weekStart
            let captureEnd = weekEnd
            let targetSlug = muscleSlug
            
            let descriptor = FetchDescriptor<SetEntry>(
                predicate: #Predicate<SetEntry> { entry in
                    entry.isComplete == true
                    && entry.isWarmup == false
                    && workingKinds.contains(entry.setTypeRaw)
                    && entry.sessionExercise?.session?.startedAt ?? .distantPast >= captureStart
                    && entry.sessionExercise?.session?.startedAt ?? .distantPast < captureEnd
                }
            )
            let entries = (try? ctx.fetch(descriptor)) ?? []
            
            // Aggregate per exercise.
            var contribByExercise: [UUID: (name: String, direct: Double, indirect: Double)] = [:]
            for entry in entries {
                guard let exercise = entry.sessionExercise?.exercise else { continue }
                let exID = exercise.id
                let stimuli = exercise.muscleStimuli ?? []
                for stim in stimuli {
                    guard stim.muscle?.slug == targetSlug else { continue }
                    var existing = contribByExercise[exID] ?? (name: exercise.canonicalName, direct: 0, indirect: 0)
                    switch stim.role {
                    case "primary":   existing.direct += stim.weight
                    case "secondary": existing.indirect += stim.weight
                    default: break
                    }
                    contribByExercise[exID] = existing
                }
            }
            
            // Sort DESC by total.
            return contribByExercise.map { (id, value) in
                ContributingExercise(
                    exerciseID: id,
                    exerciseName: value.name,
                    directSets: value.direct,
                    indirectSets: value.indirect
                )
            }.sorted { ($0.directSets + $0.indirectSets) > ($1.directSets + $1.indirectSets) }
        }
        
        private struct ContributingExercise: Identifiable {
            let exerciseID: UUID
            let exerciseName: String
            let directSets: Double
            let indirectSets: Double
            var id: UUID { exerciseID }
            
            var trailingCopy: String {
                let total = Int((directSets + indirectSets).rounded())
                let direct = Int(directSets.rounded())
                if directSets > 0 && indirectSets > 0 {
                    return "\(total) sets (\(direct) direct)"
                } else {
                    return "\(total) sets"
                }
            }
        }
    }
