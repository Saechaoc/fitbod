**ProgressHomeView layout decision (UI-SPEC § ProgressHomeView lines 130–154):** Single-column inset-grouped List with three sections (Recent / Trends / Compare). Not a grid, not a hybrid.

**Per-section composition (UI-SPEC §"Progress tab and ProgressHomeView" copy table):**

Section 1 — "Recent":
- Per-exercise cards sorted by `Session.startedAt` desc per exercise's most recent session
- Row: exercise.name + ".caption" sub-label "{n} sessions · last logged {relative-date}"
- Tap → push ExerciseProgressView(exercise:)
- Empty state: heading "No history yet", body "Log a session to start seeing your progress."

Section 2 — "Trends":
- Row "Weekly Tonnage" + sub-label "{this-week-total} kg this week · {Δ-vs-prev}" → push WeeklyTonnageView
- Row "Personal Records" + sub-label "{count} tracked across {n} exercises" → push ExercisePRsView(scope: .allExercises)
- Trophy icon `trophy.fill` (optional per UI-SPEC Asset Contract)

Section 3 — "Compare":
- Row "This week vs last week" + sub-label "{matched-exercise-count} exercises matched" → push SessionComparisonView(anchorSession: mostRecentCompletedSession)
- Empty state: "No comparable session yet — log a routine twice to compare."

**Recent section data source:** `@Query<SessionExercise>` ordered by `session?.startedAt` desc, then in-memory `Dictionary(grouping:by: \.exercise?.id)` partitioning to keep one card per exercise (most-recent SessionExercise wins). The sub-label "{n} sessions" counts distinct sessions for that exercise; "{last logged}" relative date uses `.relative(presentation: .named)`. Limit to top 10 most-recent exercises.

**Trends section data source:** A small @Query<SetEntry> filtered to the current week (this Mon → next Mon) for "this-week-total"; a second small @Query for last week for Δ. PR count via @Query<SetEntry> + PRDetector.buildTable + count of distinct (exerciseID, intent, kind, bucket) entries (acceptable at v1 scale).

**Compare section data source:** `@Query<Session>` for the most recent completed session (`completedAt != nil`, sorted desc, fetchLimit 1). Sub-label uses `SessionComparator.compare(...).filter { $0.lastWeekTopSet != nil }.count`.

**NavigationStack wiring:**
```
public struct ProgressHomeView: View {
    @Binding public var path: NavigationPath
    @Environment(\.modelContext) private var modelContext
    // @Query for recent SessionExercises, etc.

    public init(path: Binding<NavigationPath>) {
        self._path = path
    }

    public var body: some View {
        NavigationStack(path: $path) {
            List {
                recentSection
                trendsSection
                compareSection
            }
            .listStyle(.insetGrouped)
            .navigationTitle("Progress")
            .navigationDestination(for: ExerciseRoute.self) { route in
                switch route {
                case .progress(let exercise): ExerciseProgressView(exercise: exercise)
                case .prs(let exercise):      ExercisePRsView(scope: .forExercise(exercise.id))
                }
            }
            .navigationDestination(for: ProgressRoute.self) { route in
                switch route {
                case .weeklyTonnage:                WeeklyTonnageView()
                case .allPRs:                       ExercisePRsView(scope: .allExercises)
                case .sessionComparison(let s):     SessionComparisonView(anchorSession: s)
                }
            }
        }
    }
}

enum ExerciseRoute: Hashable {
    case progress(Exercise)
    case prs(Exercise)
}

enum ProgressRoute: Hashable {
    case weeklyTonnage
    case allPRs
    case sessionComparison(Session)
}
```

`Exercise` and `Session` are @Model classes which conform to Hashable automatically (planner: verify; if not, wrap in a Hashable struct keyed by id). UI-SPEC tab icon: `chart.line.uptrend.xyaxis` (replaces existing `chart.xyaxis.line` in RootView.swift line 189 — PATTERNS adjudication #3).
