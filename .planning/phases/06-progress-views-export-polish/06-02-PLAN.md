**PRTable shape (planner decision):**
```
public struct PRRecord: Sendable, Equatable {
    public let weight: Double
    public let reps: Int
    public let rpe: Double?
    public let setEntryID: UUID
    public let sessionStartedAt: Date
}

public struct PRTable: Sendable {
    // Keyed by (exerciseID, intent, kind, repBucket). RepBucket: .strength (1..6), .hypertrophy (7..10), .highRep (11+).
    // e1RM PRKind only allowed for non-.highRep buckets (suppression rule from CONTEXT D-04 / D-11).
    public func record(exerciseID: UUID, intent: Intent, kind: PRKind, bucket: RepBucket) -> PRRecord?
}

public enum RepBucket: String, CaseIterable, Sendable { case strength; case hypertrophy; case highRep }

public enum PRDetector {
    public static func buildTable(from sets: [SetEntry]) -> PRTable
    public static func check(set: SetEntry, against table: inout PRTable) -> Set<PRKind>
}
```

`check` MUTATES the table in-place when a new PR is recorded so subsequent sets in the same session compare against the updated table (D-14 + UI-SPEC § In-session PR banner flow step 5).

**PRKind canonical order (UI-SPEC § In-session PR banner copy):** weight → reps → volume → e1RM. Use `String` raw values for diff-friendliness (`"weight"`, `"reps"`, `"volume"`, `"e1rm"`).
