**CORRECTED matching rule (RESEARCH §"Schema reality vs CONTEXT" §"D-22 correction" lines 856–857):**

For each `current.exercises[i]: SessionExercise`, find the most recent prior `SessionExercise` where:
1. `prior.exercise?.id == current.exercise?.id`
2. `prior.intentRaw == current.intentRaw`
3. `prior.session?.sourceRoutineID == current.session?.sourceRoutineID` (skip when current session has nil sourceRoutineID — freestanding session, no comparable)
4. `current.session.startedAt - prior.session.startedAt < 14 days AND > 0`

NOT a per-session match. This is the load-bearing correction.

**Target signatures:**
```
public struct SessionComparisonRow: Sendable, Equatable {
    public let exerciseID: UUID
    public let exerciseName: String          // snapshot via current.exercise?.name; "" if exercise == nil
    public let thisWeekTopSet: TopSetSummary?
    public let lastWeekTopSet: TopSetSummary?
    public let deltaWeight: Double?          // nil when either side missing
    public let deltaReps: Int?
    public let deltaE1RM: Double?
}

public struct TopSetSummary: Sendable, Equatable {
    public let weight: Double
    public let reps: Int
    public let rpe: Double?
    public let e1RM: Double?                 // OneRepMax.estimate result; nil when reps > 10 or non-positive
    public let setEntryID: UUID
    public let sessionStartedAt: Date
}

public enum SessionComparator {
    /// One row per SessionExercise in `current` (sorted by orderIndex).
    /// `thisWeekTopSet` is the current session's top working set;
    /// `lastWeekTopSet` is the prior matched session's top working set per the corrected D-22 rule.
    public static func compare(
        current: Session,
        context: ModelContext,
        windowDays: Int = 14
    ) -> [SessionComparisonRow]

    /// Lower-level helper: finds the prior matching SessionExercise for a single current SessionExercise.
    public static func findPriorMatch(
        current: SessionExercise,
        currentSession: Session,
        context: ModelContext,
        windowDays: Int = 14
    ) -> SessionExercise?
}
```

**Top working set (CONTEXT D-05):** Among `sessionExercise.sets` filtered to `setType == .working && !isWarmup && isComplete && reps > 0 && weight > 0`, pick the SetEntry with the highest `OneRepMax.estimate(weight:reps:) ?? 0`. Tiebreaker 1: highest weight. Tiebreaker 2: latest completedAt.

**Predicate construction (RESEARCH Pitfall 3 + PATTERNS §Pattern C):**
```
let targetExerciseID = current.exercise?.id
let targetIntent = current.intentRaw
let targetRoutineID = currentSession.sourceRoutineID
let lowerBound = currentSession.startedAt.addingTimeInterval(-Double(windowDays) * 86_400)
let upperBound = currentSession.startedAt

var descriptor = FetchDescriptor<SessionExercise>(
    predicate: #Predicate { se in
        se.intentRaw == targetIntent
            && se.exercise?.id == targetExerciseID
            && se.session?.sourceRoutineID == targetRoutineID
            && se.session?.startedAt >= lowerBound
            && se.session?.startedAt < upperBound
    },
    sortBy: [SortDescriptor(\.session?.startedAt, order: .reverse)]
)
descriptor.fetchLimit = 1
```

**Performance budget:** One fetch per current SessionExercise. For a 6-exercise session, six fetches at session-comparison-view appear. Acceptable at v1 scale; the Phase 1 `#Index<SessionExercise>([\.intentRaw])` is the only index touched.
