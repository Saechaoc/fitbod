# Plan 04-02 — Mid-Session Swap, Add Unplanned, Tempo / Partials / Cluster Rows

> **Wave 4 / Sequence 2.** Ships the four remaining SESS-* requirements that plan `04-01` deferred to keep that plan from bloating: mid-session swap (SESS-05), add unplanned (SESS-06), optional tempo row (SESS-07), partial reps + cluster sub-reps (SESS-08). Adds `SwapExerciseSheet` + `AddUnplannedExerciseButton` (both reusing Phase 1's `ExerciseLibraryView(onSelect:)` picker init from plan `03-02`). Adds `TempoEntryRow` + `PartialRepsRow` + `ClusterSubRepChipRow` — three opt-in row types that render conditionally based on the snapshotted toggles + set type. The long-press header menu on `SessionExerciseCard` routes "Swap Exercise…" + "Remove from Session" actions.

## Goal

Long-press an exercise card header → menu: "Swap Exercise…" (presents `SwapExerciseSheet` with embedded `ExerciseLibraryView(onSelect:)`) + "Remove from Session" (destructive confirmation). Bottom of the session list: "+ Add Exercise" button (`AddUnplannedExerciseButton`) → opens picker → appends a new `SessionExercise` row to the active session. Neither swap nor add touches the source `Routine` — PITFALLS-doc #1.

Opt-in row rendering on `SessionExerciseCard`:
- `TempoEntryRow` renders when `sessionExercise.routineExercise?.tracksTempo` would be true — BUT `SessionExercise` has no routineExercise ref. The snapshotted `tempo: String?` field on `SessionExercise` from Phase 1 is the seed; the toggle's runtime state is "if tempo was set on the source RE." Since `SessionExercise` snapshots `tempo: String?` (the prescribed value), and the per-set actual tempo lives on `SetEntry.tempoActual: String?`, the toggle for "render this row" can be derived from whether the snapshotted `tempo` is non-nil OR added as a new SchemaV2 field on `SessionExercise` — but plan `00-01` did not add such a field. Resolution: the `tracksTempo` flag from `RoutineExercise` IS snapshotted via a new SchemaV2 field on `SessionExercise.tracksTempo: Bool` that this plan adds to `SessionFactory.start`.

Wait — plan `00-01` ALREADY added `tracksTempo: Bool` on `RoutineExercise` only, not on `SessionExercise`. Per the snapshot pattern, every routine-side toggle that affects session-side rendering MUST be snapshotted. **Schema correction**: this plan adds two additive fields on `SessionExercise`: `tracksTempo: Bool = false` and `tracksPartialReps: Bool = false`. These are additive, default-valued — lightweight migration extends from V2 → still-V2 (since SchemaV2 was just landed in plan `00-02`, adding fields BEFORE V2 ships to a real user means we update SchemaV2.swift directly with the new fields; no V3 needed).

This means the executor of this plan modifies:
- `fitbod/Models/SessionExercise.swift` — add `tracksTempo: Bool = false` and `tracksPartialReps: Bool = false`.
- `fitbod/Sessions/SessionFactory.swift` (plan `01-01`) — snapshot these two fields in the deep-copy.

(This is the kind of mid-wave schema correction that the additive-only design pattern enables — no V3 migration needed because V2 hasn't shipped to any user yet. The added fields default to `false`; existing planned-Session rows just keep the defaults.)

## Requirements Covered

- **SESS-05** (mid-session swap without mutating routine template) — `SwapExerciseSheet` mutates `SessionExercise.exercise` only; the source `RoutineExercise.exercise` is untouched.
- **SESS-06** (add unplanned exercise mid-session) — `AddUnplannedExerciseButton` appends a new `SessionExercise` to `session.exercises` only; the source `Routine.exercises` is untouched.
- **SESS-07** (optional 4-field tempo per set) — `TempoEntryRow` rendered when `SessionExercise.tracksTempo == true` (snapshotted from the source `RoutineExercise.tracksTempo`).
- **SESS-08** (partial reps + cluster sub-reps) — `PartialRepsRow` rendered when `SessionExercise.tracksPartialReps == true`; `ClusterSubRepChipRow` rendered when the set's `setType == .restPause`.

import SwiftUI

public struct SwapExerciseSheet: View {
    @Environment(\.modelContext) private var ctx
    @Environment(\.dismiss) private var dismiss

    @Bindable public var sessionExercise: SessionExercise

    public init(sessionExercise: SessionExercise) {
        self.sessionExercise = sessionExercise
    }

    public var body: some View {
        NavigationStack {
            VStack(spacing: 0) {
                ExerciseLibraryView(onSelect: { exercise in
                    // SESS-05 — mutate SessionExercise.exercise ONLY.
                    sessionExercise.exercise = exercise
                    // Reset planned weight if any future sets exist with
                    // the old hint. Leave logged sets alone (PITFALLS-doc
                    // #1 — historical data is immutable).
                    let pendingSets = (sessionExercise.sets ?? [])
                        .filter { !$0.isComplete }
                    let hint = PreviousMatchingIntent.fetchTopWorkingSet(
                        exerciseID: exercise.id,
                        intentRaw: sessionExercise.intentRaw,
                        context: ctx
                    )?.weight ?? 0
                    for set in pendingSets {
                        set.weight = hint
                        set.reps = 0
                        set.rpe = nil
                    }
                    try? ctx.save()
                    dismiss()
                })
                Text("This swap applies to this session only. The routine template will not change.")   // UI-SPEC verbatim
                    .font(.caption)
                    .foregroundStyle(.secondary)
                    .multilineTextAlignment(.center)
                    .padding(16)
            }
            .navigationTitle("Swap Exercise")                                  // UI-SPEC verbatim
            .toolbar {
                ToolbarItem(placement: .topBarLeading) {
                    Button("Cancel") { dismiss() }
                }
            }
        }
    }
}

import SwiftUI

public struct AddUnplannedExerciseButton: View {
    @Environment(\.modelContext) private var ctx
    @Bindable public var session: Session
    @State private var presentingPicker = false

    public init(session: Session) {
        self.session = session
    }

    public var body: some View {
        Button {
            presentingPicker = true
        } label: {
            HStack(spacing: 4) {
                Image(systemName: "plus.circle")
                Text("Add Exercise")                                            // UI-SPEC verbatim
            }
            .foregroundStyle(Color.accentColor)
        }
        .buttonStyle(.plain)
        .sheet(isPresented: $presentingPicker) {
            NavigationStack {
                ExerciseLibraryView(onSelect: { exercise in
                    append(exercise: exercise)
                    presentingPicker = false
                })
                .navigationTitle("Add Exercise")
                .toolbar {
                    ToolbarItem(placement: .topBarLeading) {
                        Button("Cancel") { presentingPicker = false }
                    }
                }
            }
        }
    }

    private func append(exercise: Exercise) {
        // SESS-06 — append SessionExercise to the active session only.
        // The source routine is untouched.
        let se = SessionExercise()
        se.session = session
        se.exercise = exercise
        se.orderIndex = (session.exercises ?? []).count
        // Default prescription (no source RoutineExercise to snapshot
        // from). Intent inherits from any prior matching-intent session
        // for this exercise; otherwise defaults to hypertrophy.
        se.intentRaw = Intent.hypertrophy.rawValue
        se.targetSets = 3
        se.targetRepsLow = 8
        se.targetRepsHigh = 12
        se.prescribedRestSeconds = exercise.mechanic == .compound ? 180 : 90
        ctx.insert(se)

        // Seed 3 planned sets with a previous-matching-intent hint.
        let hint = PreviousMatchingIntent.fetchTopWorkingSet(
            exerciseID: exercise.id,
            intentRaw: se.intentRaw,
            context: ctx
        )?.weight ?? 0
        for i in 0..<3 {
            let entry = SetEntry()
            entry.sessionExercise = se
            entry.orderIndex = i
            entry.weight = hint
            entry.reps = 0
            entry.setTypeRaw = SetType.working.rawValue
            entry.isComplete = false
            entry.completedAt = .now
            ctx.insert(entry)
        }
        try? ctx.save()
    }
}

import SwiftUI

public struct TempoEntryRow: View {
    @Bindable public var entry: SetEntry

    public init(entry: SetEntry) {
        self.entry = entry
    }

    public var body: some View {
        HStack(spacing: 8) {
            Text("Tempo").font(.caption).foregroundStyle(.secondary)            // UI-SPEC verbatim
            ForEach(0..<4, id: \.self) { idx in
                TextField(label(idx), text: Binding(
                    get: { components[idx] },
                    set: { components[idx] = $0; persistTempo() }
                ))
                .keyboardType(.numberPad)
                .frame(width: 32)
            }
            Spacer()
        }
        .font(.caption)
    }

    private var components: [String] {
        get {
            let parts = (entry.tempoActual ?? "").split(separator: "-").map { String($0) }
            return (0..<4).map { i in i < parts.count ? parts[i] : "" }
        }
        nonmutating set {
            let nonEmpty = newValue.filter { !$0.isEmpty }
            entry.tempoActual = nonEmpty.isEmpty ? nil : newValue.joined(separator: "-")
        }
    }

    private func label(_ idx: Int) -> String {
        ["Ecc", "Bot", "Con", "Top"][idx]                                       // UI-SPEC verbatim labels
    }

    private func persistTempo() {
        // No-op — the @Bindable wrap + .components setter persists.
    }
}

## Anti-Patterns to Avoid

- Do NOT mutate `RoutineExercise.exercise` during swap. RESEARCH § Anti-Patterns: swap mutates `SessionExercise.exercise` only. The source routine is immutable from the moment `SessionFactory.start` runs.
- Do NOT mutate source `Routine.exercises` during add-unplanned. SESS-06 contract: appended SE rows live on the active `Session.exercises` only.
- Do NOT skip the `pendingSets` re-seed on swap. Pending (un-committed) sets carry the OLD exercise's previous-hint weight; the swap should update them to the new exercise's previous hint. Committed sets stay (the snapshot is immutable).
- Do NOT pre-add a `SessionExerciseTracksFields` migration step. The added fields are still SchemaV2 — they slot into the in-flight SchemaV2 before V2 ever ships. Lightweight migration handles the default-valued additive fields automatically.
- Do NOT render `TempoEntryRow` / `PartialRepsRow` / `ClusterSubRepChipRow` unconditionally. UI-SPEC § Tempo / Partial reps / Cluster: each is opt-in per exercise; cluster is opt-in per SET type.
- Do NOT use `SessionExercise.tempo` as the toggle gate for `TempoEntryRow`. `tempo: String?` is the prescribed tempo string (e.g. "3-1-1-0"); the toggle gate is the explicit `tracksTempo: Bool` snapshotted field.
