# Plan 03-01 — Routines Tab, Folders, and Resume Workout Banner

> **Wave 3 / Sequence 1.** Replaces the `PlaceholderTabView(phaseNumber: 2)` placeholder on the Routines tab with the real `RoutinesListView` — sectioned `List` grouped by `RoutineFolder`, with the "+" toolbar Menu (New Routine / New Folder), swipe actions (Start Workout / Delete / Duplicate), and the empty state. Also ships `ResumeWorkoutBanner` (top-of-tab card surfacing the active session if one exists) and the active-session-conflict guard for "Start Workout" taps. Closes ROUTINE-06 (folders) + the user-visible half of SESS-04's "Resume Workout" surfacing.

## Goal

The user opens the app, taps Routines, and sees their routines grouped by folder. Empty state on first launch nudges them toward "New Routine." The "+" toolbar opens a Menu with "New Routine" (pushes `RoutineBuilderView(draft: RoutineDraft())` — built in plan `03-02`) and "New Folder" (presents `NewFolderSheet`). Swiping a routine row leading reveals "Start Workout" (accent fill, white label, `play.fill` glyph); swiping trailing reveals "Delete" (destructive) + "Duplicate" (secondary). Long-press the row → full context menu: "Start Workout" / "Duplicate" / "Move…" / "Edit" / "Delete". Tap the row → push `RoutineBuilderView(routine:)` in edit mode.

Active-session safety: `handleStartTap(routine:)` queries `#Predicate<Session> { $0.completedAt == nil }`. If non-empty, surface the UI-SPEC "Workout in Progress" alert (no new session started). The `ResumeWorkoutBanner` uses the same query reactively.

## Requirements Covered

- **ROUTINE-06** (`User can duplicate routines and organize them into folders`) — the folders half + the duplicate action stub (the actual deep-copy logic lives in plan `03-03`, which owns "RoutineDuplicator"; this plan wires the menu item).
- **SESS-04** (rest timer / resume workout) — the user-visible "Resume Workout: {routineSnapshotName}" surfacing on app launch / Routines tab.

/// Routines tab body — sectioned by `RoutineFolder`, with a sticky
/// search bar (only renders when ≥1 routine exists), the "+" toolbar
/// Menu (New Routine / New Folder), and the empty state.
///
/// Mirrors the Phase 1 ExerciseLibraryView outer/inner split pattern:
/// the outer view owns ephemeral state (search text, presented sheets,
/// active-conflict alert routing); the inner @Query<Routine> +
/// @Query<RoutineFolder> + @Query<Session>-with-active-filter handle the
/// reactive data.
public struct RoutinesListView: View {
    @Environment(\.modelContext) private var ctx

    @Query(sort: \RoutineFolder.sortOrder) private var folders: [RoutineFolder]
    @Query(sort: [SortDescriptor(\Routine.name)]) private var routines: [Routine]
    @Query(filter: #Predicate<Session> { $0.completedAt == nil })
    private var activeSessions: [Session]

    @State private var searchText: String = ""
    @State private var presentingNewFolder = false
    @State private var presentingMoveSheet: Routine? = nil
    @State private var presentingNewRoutine = false
    @State private var conflictRoutine: Routine? = nil
    @State private var deleteConfirmRoutine: Routine? = nil
    @State private var deleteConfirmFolder: RoutineFolder? = nil

    public init() {}

    public var body: some View {
        NavigationStack {
            content
                .navigationTitle("Routines")                                  // UI-SPEC verbatim
                .toolbar {
                    ToolbarItem(placement: .topBarTrailing) {
                        Menu {
                            Button("New Routine") {                            // UI-SPEC verbatim
                                presentingNewRoutine = true
                            }
                            Button("New Folder") {                             // UI-SPEC verbatim
                                presentingNewFolder = true
                            }
                        } label: {
                            Label("Add routine or folder", systemImage: "plus") // UI-SPEC a11y
                                .labelStyle(.iconOnly)
                        }
                        .accessibilityLabel("Add routine or folder")          // UI-SPEC verbatim a11y
                    }
                }
                .sheet(isPresented: $presentingNewFolder) {
                    NewFolderSheet(draft: RoutineFolderDraft())
                }
                .sheet(item: $presentingMoveSheet) { routine in
                    MoveRoutineSheet(routine: routine, folders: folders)
                }
                .sheet(isPresented: $presentingNewRoutine) {
                    // Plan 03-02 wire — RoutineBuilderView with a fresh
                    // draft, wrapped in a NavigationStack to own its
                    // own toolbar.
                    Text("New Routine — Plan 03-02 fills this in.")           // INTERIM — replaced by plan 03-02
                }
                .alert(
                    "Workout in Progress",                                    // UI-SPEC verbatim
                    isPresented: Binding(
                        get: { conflictRoutine != nil },
                        set: { if !$0 { conflictRoutine = nil } }
                    )
                ) {
                    Button("Resume Workout") {
                        conflictRoutine = nil
                        // Navigation to SessionLoggerView happens via the
                        // resume banner mounted above; the alert just
                        // dismisses.
                    }
                    Button("Discard", role: .destructive) {
                        if let active = activeSessions.first {
                            ctx.delete(active)
                            try? ctx.save()
                        }
                        conflictRoutine = nil
                    }
                    Button("Cancel", role: .cancel) {
                        conflictRoutine = nil
                    }
                } message: {
                    Text("Finish or discard the current workout before starting a new one.")  // UI-SPEC verbatim
                }
                .confirmationDialog(
                    deleteConfirmFolder.map { "Delete \"\($0.name)\"?" } ?? "Delete?",
                    isPresented: Binding(
                        get: { deleteConfirmFolder != nil },
                        set: { if !$0 { deleteConfirmFolder = nil } }
                    ),
                    titleVisibility: .visible
                ) {
                    Button("Delete", role: .destructive) {
                        if let folder = deleteConfirmFolder {
                            handleFolderDelete(folder)
                        }
                        deleteConfirmFolder = nil
                    }
                    Button("Cancel", role: .cancel) {
                        deleteConfirmFolder = nil
                    }
                } message: {
                    Text("The folder will be removed.")                       // UI-SPEC verbatim
                }
        }
    }

    @ViewBuilder private var content: some View {
        if routines.isEmpty && folders.isEmpty {
            emptyState
        } else {
            List {
                ResumeWorkoutBanner(
                    onResume: { _ in
                        // TODO plan 04-01: navigate to SessionLoggerView via NavigationPath
                    },
                    onDiscard: { session in
                        ctx.delete(session)
                        try? ctx.save()
                    }
                )
                .listRowInsets(EdgeInsets())
                .listRowBackground(Color.clear)

                ForEach(sectionsForRendering, id: \.id) { section in
                    Section(section.title) {
                        ForEach(section.routines) { routine in
                            RoutineRow(
                                routine: routine,
                                onTap: { /* push RoutineBuilderView edit mode — plan 03-02 */ },
                                onStart: { handleStartTap(routine: $0) },
                                onDuplicate: { handleDuplicate(routine: $0) },
                                onMove: { presentingMoveSheet = $0 },
                                onDelete: { handleDelete(routine: $0) }
                            )
                        }
                    }
                }
            }
            .listStyle(.insetGrouped)
        }
    }

    private var sectionsForRendering: [RoutineSection] {
        let unfiled = routines.filter { $0.folderID == nil }
        let folderSections = folders.map { folder -> RoutineSection in
            let folderRoutines = routines.filter { $0.folderID == folder.id }
            return RoutineSection(id: folder.id, title: folder.name, routines: folderRoutines)
        }
        var out = [RoutineSection]()
        if !unfiled.isEmpty {
            out.append(RoutineSection(id: UUID(), title: "Unfiled", routines: unfiled))   // UI-SPEC verbatim
        }
        out.append(contentsOf: folderSections)
        return out
    }

    private struct RoutineSection: Identifiable {
        let id: UUID
        let title: String
        let routines: [Routine]
    }

    @ViewBuilder private var emptyState: some View {
        VStack(spacing: 16) {
            Spacer().frame(height: 48)                                        // UI-SPEC 3xl top spacer
            Text("No routines yet")                                            // UI-SPEC verbatim
                .font(.title2)
                .fontWeight(.semibold)
            Text("Build a routine to start logging workouts.")                 // UI-SPEC verbatim
                .font(.body)
                .foregroundStyle(.secondary)
                .multilineTextAlignment(.center)
            Button("New Routine") {                                            // UI-SPEC verbatim accent text button
                presentingNewRoutine = true
            }
            .foregroundStyle(Color.accentColor)
            Spacer()
        }
        .padding(.horizontal, 32)
    }

    private func handleStartTap(routine: Routine) {
        if !activeSessions.isEmpty {
            conflictRoutine = routine
            return
        }
        do {
            _ = try SessionFactory.start(routine: routine, on: .now, context: ctx)
            // Plan 04-01 — navigate to SessionLoggerView; deferred here.
        } catch SessionFactoryError.activeSessionAlreadyExists {
            conflictRoutine = routine
        } catch {
            // Could surface "Couldn't Start Workout" — plan 04-01 polish.
        }
    }

    private func handleDuplicate(routine: Routine) {
        // Plan 03-03 wires `RoutineDuplicator.duplicate(routine:context:)` —
        // this plan calls the entry-point but the duplicator file lands
        // in plan 03-03. For now the call site is a stub:
        // TODO plan 03-03: RoutineDuplicator.duplicate(routine: routine, context: ctx)
    }

    private func handleDelete(routine: Routine) {
        // ROUTINE-07 / PITFALLS-doc #1 — deleting a routine MUST NOT
        // affect historical Sessions because Session.sourceRoutineID is
        // a soft ref. The SwiftData cascade on Routine deletes its
        // RoutineExercise + setOverrides + supersets (the SupersetGroup
        // soft refs are explicitly zeroed via a query-and-delete pass).
        let id = routine.id
        let supersets = (try? ctx.fetch(FetchDescriptor<SupersetGroup>(
            predicate: #Predicate { $0.routineID == id }
        ))) ?? []
        for sg in supersets {
            ctx.delete(sg)
        }
        ctx.delete(routine)
        try? ctx.save()
    }

    private func handleFolderDelete(_ folder: RoutineFolder) {
        // Soft ref design: deleting a folder MUST NOT cascade routines.
        // Routines whose `folderID == folder.id` are re-mapped to nil
        // (Unfiled) before the folder is deleted.
        let folderID = folder.id
        let affected = (try? ctx.fetch(FetchDescriptor<Routine>(
            predicate: #Predicate { $0.folderID == folderID }
        ))) ?? []
        for routine in affected {
            routine.folderID = nil
        }
        ctx.delete(folder)
        try? ctx.save()
    }
}

import SwiftUI

public struct RoutineRow: View {
    public let routine: Routine
    public let onTap: (Routine) -> Void
    public let onStart: (Routine) -> Void
    public let onDuplicate: (Routine) -> Void
    public let onMove: (Routine) -> Void
    public let onDelete: (Routine) -> Void

    public init(
        routine: Routine,
        onTap: @escaping (Routine) -> Void,
        onStart: @escaping (Routine) -> Void,
        onDuplicate: @escaping (Routine) -> Void,
        onMove: @escaping (Routine) -> Void,
        onDelete: @escaping (Routine) -> Void
    ) {
        self.routine = routine
        self.onTap = onTap
        self.onStart = onStart
        self.onDuplicate = onDuplicate
        self.onMove = onMove
        self.onDelete = onDelete
    }

    public var body: some View {
        VStack(alignment: .leading, spacing: 4) {
            Text(routine.name)
                .font(.body)
            Text(subtitle)
                .font(.caption)
                .foregroundStyle(.secondary)
        }
        .contentShape(Rectangle())
        .onTapGesture { onTap(routine) }
        .swipeActions(edge: .leading) {
            Button {
                onStart(routine)
            } label: {
                Label("Start Workout", systemImage: "play.fill")              // UI-SPEC verbatim
            }
            .tint(Color.accentColor)                                          // UI-SPEC accent #13
        }
        .swipeActions(edge: .trailing) {
            Button(role: .destructive) {
                onDelete(routine)
            } label: {
                Label("Delete", systemImage: "trash")
            }
            Button {
                onDuplicate(routine)
            } label: {
                Label("Duplicate", systemImage: "doc.on.doc")
            }
            .tint(.gray)
        }
        .contextMenu {
            Button("Start Workout") { onStart(routine) }                       // UI-SPEC verbatim
                .foregroundStyle(Color.accentColor)
            Button("Duplicate") { onDuplicate(routine) }                       // UI-SPEC verbatim
            Button("Move…") { onMove(routine) }                                // UI-SPEC verbatim
            Button("Edit") { onTap(routine) }                                  // UI-SPEC verbatim
            Divider()
            Button("Delete", role: .destructive) { onDelete(routine) }         // UI-SPEC verbatim
        }
    }

    private var subtitle: String {
        let count = (routine.exercises ?? []).count
        return "\(count) exercises"                                            // UI-SPEC subtitle format
    }
}

import SwiftUI
import SwiftData

public struct ResumeWorkoutBanner: View {
    @Environment(\.modelContext) private var ctx
    @Query(filter: #Predicate<Session> { $0.completedAt == nil })
    private var activeSessions: [Session]

    public let onResume: (Session) -> Void
    public let onDiscard: (Session) -> Void

    @State private var discardConfirm = false

    public init(onResume: @escaping (Session) -> Void, onDiscard: @escaping (Session) -> Void) {
        self.onResume = onResume
        self.onDiscard = onDiscard
    }

    public var body: some View {
        if let active = activeSessions.first {
            HStack(spacing: 12) {                                              // UI-SPEC md
                Image(systemName: "arrow.uturn.backward.circle")               // UI-SPEC Asset Contract
                    .foregroundStyle(Color.accentColor)
                VStack(alignment: .leading) {
                    Text("Resume Workout: \(active.routineSnapshotName)")      // UI-SPEC verbatim
                        .font(.headline)
                }
                Spacer()
                Button("Resume") { onResume(active) }                          // UI-SPEC verbatim accent text
                    .foregroundStyle(Color.accentColor)
                Button("Discard", role: .destructive) {                        // UI-SPEC verbatim destructive text
                    discardConfirm = true
                }
            }
            .padding(16)                                                       // UI-SPEC lg
            .background(Color(.secondarySystemGroupedBackground))
            .clipShape(RoundedRectangle(cornerRadius: 12))
            .padding(.horizontal, 16)
            .alert(
                "Discard active workout?",                                     // UI-SPEC verbatim
                isPresented: $discardConfirm
            ) {
                Button("Discard", role: .destructive) {
                    onDiscard(active)
                    discardConfirm = false
                }
                Button("Cancel", role: .cancel) {
                    discardConfirm = false
                }
            }
        }
    }
}

@MainActor
@Suite("ActiveSessionConflict")
struct ActiveSessionConflictTests {

    private func makeContext() throws -> ModelContext {
        let schema = Schema(SchemaV2.models)
        let config = ModelConfiguration(schema: schema, isStoredInMemoryOnly: true)
        let container = try ModelContainer(
            for: schema,
            migrationPlan: FitbodSchemaMigrationPlan.self,
            configurations: config
        )
        return ModelContext(container)
    }

    private func makeRoutine(ctx: ModelContext) throws -> Routine {
        let routine = Routine()
        routine.name = "Test"
        ctx.insert(routine)

        let re = RoutineExercise()
        re.routine = routine
        re.targetSets = 3
        let ex = Exercise.previewSample(name: "Bench")
        ctx.insert(ex)
        re.exercise = ex
        ctx.insert(re)
        try ctx.save()
        return routine
    }

    @Test("firstStartSucceeds — no active session exists")
    func firstStartSucceeds() throws {
        let ctx = try makeContext()
        let routine = try makeRoutine(ctx: ctx)
        let session = try SessionFactory.start(routine: routine, on: .now, context: ctx)
        #expect(session.completedAt == nil)
    }

    @Test("secondStartThrowsActiveSessionAlreadyExists")
    func secondStartThrowsActiveSessionAlreadyExists() throws {
        let ctx = try makeContext()
        let routine = try makeRoutine(ctx: ctx)
        _ = try SessionFactory.start(routine: routine, on: .now, context: ctx)

        do {
            _ = try SessionFactory.start(routine: routine, on: .now, context: ctx)
            Issue.record("Expected activeSessionAlreadyExists")
        } catch SessionFactoryError.activeSessionAlreadyExists {
            // expected
        }
    }

    @Test("startAfterFinishingPriorSessionSucceeds")
    func startAfterFinishingPriorSessionSucceeds() throws {
        let ctx = try makeContext()
        let routine = try makeRoutine(ctx: ctx)
        let first = try SessionFactory.start(routine: routine, on: .now, context: ctx)
        first.completedAt = .now
        try ctx.save()

        let second = try SessionFactory.start(routine: routine, on: .now, context: ctx)
        #expect(second.id != first.id)
    }

    @Test("startAfterDiscardingPriorSessionSucceeds")
    func startAfterDiscardingPriorSessionSucceeds() throws {
        let ctx = try makeContext()
        let routine = try makeRoutine(ctx: ctx)
        let first = try SessionFactory.start(routine: routine, on: .now, context: ctx)
        ctx.delete(first)
        try ctx.save()

        let second = try SessionFactory.start(routine: routine, on: .now, context: ctx)
        #expect(second.completedAt == nil)
    }
}

## Anti-Patterns to Avoid

- Do NOT use a SwiftData `@Relationship` for `Routine ↔ RoutineFolder`. Soft `UUID?` ref by design (CONTEXT.md Area 6). Folder deletion does NOT cascade routines.
- Do NOT call `SessionFactory.start(...)` from a swipe action without first checking `activeSessions.isEmpty`. The factory itself throws on conflict, but the user-visible alert should be presented from the caller (not bubbled up from a generic error).
- Do NOT mount `ResumeWorkoutBanner` inside the `NavigationStack`'s content view — mount it as the first row of the `List` (using `listRowInsets(EdgeInsets())` + `listRowBackground(Color.clear)` to make it visually float).
- Do NOT push a `RoutineBuilderView` destination in this plan. Plan `03-02` ships the destination; this plan leaves the row-tap as a no-op stub with a TODO comment.
- Do NOT wire the "Duplicate" menu item to inline `Routine` cloning. Plan `03-03` ships `RoutineDuplicator.duplicate(...)`; this plan stubs the call.
- Do NOT skip the empty-state CTA "New Routine" button — UI-SPEC § Empty states locks the copy + accent treatment. Without the CTA, the empty state is a dead-end screen.
- Do NOT add a parallel `RoutinesListViewModel` class. FOUND-06 / MV-VM-lite — views consume `@Query` and `@Bindable` directly. The mutable folder-draft state lives on `RoutineFolderDraft` (`@Observable`), which is an ephemeral mutable wrapper, not a ViewModel.
- Do NOT issue more than one `try ctx.save()` per logical user action. The folder-rename / delete handler should batch all mutations + a single `save()` at the end.
