# Plan 04-03 — Session Notes (Workout / Pinned Per-Exercise / Per-Set)

> **Wave 4 / Sequence 3.** Closes the SESS-02 notes half + SESS-11 (workout-level + pinned per-exercise notes). Adds `WorkoutNotesSheet` (header "Notes" chip → editor for `session.notes`), `PerSetNoteSheet` (set-row notes button → editor for `setEntry.notes`), `PinnedNoteSheet` (long-press exercise → "Edit Pinned Note" → editor for `sessionExercise.pinnedNote`), and `PinnedNoteCapsule` (inline yellow capsule on the exercise card when `pinnedNote != nil`). Three small sheets + one inline rendering view + a few wire-ups in plan `04-01` / `04-02`'s files.

## Goal

Three note surfaces, all simple `TextField`-driven sheets:

1. **Workout-level notes** — the header chip "Notes" in `SessionLoggerView` (already wired in plan `04-01` but the sheet body was a TODO). This plan ships `WorkoutNotesSheet` binding `session.notes`.

2. **Pinned per-exercise notes** — inline yellow capsule on `SessionExerciseCard` showing `pinnedNote` when non-nil. Tapping the capsule opens `PinnedNoteSheet` editing `sessionExercise.pinnedNote`. Long-press menu adds "Edit Pinned Note" entry (already stubbed in plan `04-02`).

3. **Per-set notes** — small `Image(systemName: "square.and.pencil")` button on `SetRow`'s trailing edge (BEFORE the completion checkmark). Tap opens `PerSetNoteSheet` binding `entry.notes`.

## Requirements Covered

- **SESS-02** (per-set form notes) — the per-set note button + `PerSetNoteSheet`.
- **SESS-11** (workout-level + pinned per-exercise notes inline) — `WorkoutNotesSheet` + `PinnedNoteCapsule` + `PinnedNoteSheet`.

public struct PinnedNoteCapsule: View {

                    .font(.caption)

            .padding(.vertical, 4)

        }
        .buttonStyle(.plain)
        .accessibilityLabel("Pinned note: \(note)")                            // UI-SPEC verbatim a11y

## Anti-Patterns to Avoid

- Do NOT store the notes draft on a separate `@Observable` wrapper. The sheets bind directly to the `@Model` row via `@Bindable`; mutations write through immediately. The "Save" button on `PinnedNoteSheet` is just a dismiss — the field already persisted via the SwiftData implicit save trigger.
- Do NOT use `Color.yellow` for the pinned-note capsule background. UI-SPEC verbatim: `Color(.systemYellow).opacity(0.15)` (the asset-catalog system yellow, not the SwiftUI literal).
- Do NOT make the per-set note button persistently visible at full opacity for empty notes. The accent foreground when `entry.notes != nil` is the visual signal; empty-notes buttons are secondary-label tinted.
- Do NOT add a Save button to `WorkoutNotesSheet`. UI-SPEC verbatim: only "Done" (top-bar trailing). The binding persists on text change.
- Do NOT include a workout-level notes editor in `SessionLoggerView.body` directly. The notes editor is sheet-presented to keep the main logger surface clean.
- Do NOT trigger a fresh rest-timer notification on note edit. The notes sheets are pure UI; no interaction with `RestTimerEngine`.
