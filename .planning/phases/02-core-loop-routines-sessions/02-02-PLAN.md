# Plan 02-02 — Rest Timer Live Activity (Widget Extension + ActivityKit)

> **Wave 2 / Sequence 2.** Adds the `FitbodWidgets` Widget Extension target hosting `RestTimerLiveActivity: Widget` (the ActivityKit `ActivityConfiguration` declaring lock-screen + Dynamic Island compact / minimal / expanded presentations). Wires `RestTimerActivityController` on the main-app side to call `Activity.request` / `update` / `end` lifecycle methods with ±15s debouncing (RESEARCH §6 Pitfall 9). Closes the Live Activity half of SESS-04. Plans `02-01` ships the engine + notification half; this plan ships the lock-screen card + Dynamic Island half; plan `02-03` wires them together into a single observable façade and renders the in-app overlay.

## Goal

Live Activity is the only iOS-supported way to put the rest timer countdown on the lock screen and Dynamic Island (RESEARCH § Don't Hand-Roll). Apple gates `RestTimerLiveActivity: Widget` to a Widget Extension target, so this plan adds the target and the entitlement plumbing. The main-app side gets a thin `RestTimerActivityController` that owns the `Activity<RestTimerAttributes>` reference + a 200ms debounce on `update(...)` (Pitfall 9: spam ±15s would trip the rate limiter).

Per RESEARCH § Pitfall 3: every call to `Activity.request(...)` is wrapped in `do/catch` and silently nils the activity ref on failure (simulator / pre-Pro iPhone / user-disabled Live Activities). The in-app overlay still works.

## Requirements Covered

- **SESS-04** (rest timer Live Activity / Dynamic Island while running) — the lock-screen + Dynamic Island half. The engine + notification half is plan `02-01`; the in-app overlay + integration with `RestTimerEngine` is plan `02-03`.

### Add Widget Extension target (via Xcode New Target → Widget Extension)

This is the one place in Phase 2 where the pbxproj is non-trivial. PBXFileSystemSynchronizedRootGroup (the Phase 1 auto-discovery pattern) handles file adds within an existing target, but a **new target** requires either:

(b) A direct pbxproj edit adding `PBXNativeTarget`, `PBXBuildFile`, `PBXSourcesBuildPhase`, `PBXResourcesBuildPhase`, `PBXFrameworksBuildPhase`, `XCConfigurationList`, and the bundle ID + entitlement settings.

Per the orchestrator's runtime constraint (`/gsd-autonomous --auto`, no user available), the executor **must** prefer the pbxproj edit. The reference shape:

- **Target name:** `FitbodWidgets`
- **Bundle ID:** `com.fitbod.app.FitbodWidgets` (matches the main app's `com.fitbod.app` + `.FitbodWidgets`)
- **Deployment target:** iOS 18.0 (matches main app)
- **Info.plist:** see below; declares `NSExtension` with `NSExtensionPointIdentifier = com.apple.widgetkit-extension`
- **Entitlements:** see `FitbodWidgets.entitlements` below
- **Main interface:** none (widget targets use `@main` on the bundle struct)

### Create — `fitbod/Sessions/RestTimer/RestTimerAttributes.swift`

Per RESEARCH § Pattern 3 / Code Example 3. **Must compile in both the main app target AND the widget extension target.** Membership: BOTH targets (set via "Target Membership" in Xcode file inspector, or via the pbxproj `fileRef` shared across both `PBXSourcesBuildPhase` entries).

```
import ActivityKit
import Foundation

@available(iOS 16.1, *)
public struct RestTimerAttributes: ActivityAttributes {
    public struct ContentState: Codable, Hashable {
        /// Wall-clock start time. Both the main app and the widget compute
        /// elapsed off `Date.now`. Updates are pushed when the user
        /// presses ±15s; `startedAt` does NOT move (matches the
        /// RestTimerEngine semantics).
        public var startedAt: Date
        /// Total rest in seconds. UI computes remaining = target - elapsed.
        /// Set to 0 to signal "stopped" to the lock-screen card before
        /// dismissal.
        public var targetSeconds: Int

        public init(startedAt: Date, targetSeconds: Int) {
            self.startedAt = startedAt
            self.targetSeconds = targetSeconds
        }
    }

    /// Static — set once at .request() time, never updates.
    public var sessionStartedAt: Date
    public var exerciseName: String

    public init(sessionStartedAt: Date, exerciseName: String) {
        self.sessionStartedAt = sessionStartedAt
        self.exerciseName = exerciseName
    }
}
```

### Create — `fitbod/Sessions/RestTimer/RestTimerActivityController.swift`

Main-app side. Owned by `RestTimerEngine` in plan `02-03`; this plan ships the controller as a standalone testable type.

```
import Foundation
import ActivityKit

/// Abstraction over ActivityKit for unit-testability and for the silent-
/// fallback contract (RESEARCH §6 Pitfall 3).
@MainActor
public protocol RestTimerActivityControlling {
    func start(startedAt: Date, targetSeconds: Int, exerciseName: String)
    func update(startedAt: Date, targetSeconds: Int)
    func end()
}

@available(iOS 16.1, *)
@MainActor
public final class RestTimerActivityController: RestTimerActivityControlling {

    /// The running activity, when one was successfully started.
    /// `nil` on simulator / pre-Pro / user-disabled / Apple-rate-limited.
    private var activity: Activity<RestTimerAttributes>?

    /// Debounce window for `update(...)` to respect ActivityKit's
    /// documented rate limits (RESEARCH §6 Pitfall 9). Spam ±15s
    /// coalesces into a single update after the window.
    private let debounceMillis: Int

    /// Pending debounce task — cancelled if a new update arrives.
    private var debounceTask: Task<Void, Never>?

    public init(debounceMillis: Int = 200) {
        self.debounceMillis = debounceMillis
    }

    public func start(startedAt: Date, targetSeconds: Int, exerciseName: String) {
        // RESEARCH §6 Pitfall 3 — guard with areActivitiesEnabled and
        // silently fall back on any failure.
        let authInfo = ActivityAuthorizationInfo()
        guard authInfo.areActivitiesEnabled else { return }

        // If an activity is already running (defensive — engine should
        // call end() first), end the prior one synchronously before
        // requesting a new one.
        if activity != nil {
            end()
        }

        let attributes = RestTimerAttributes(
            sessionStartedAt: startedAt,
            exerciseName: exerciseName
        )
        let state = RestTimerAttributes.ContentState(
            startedAt: startedAt,
            targetSeconds: targetSeconds
        )
        let content = ActivityContent(
            state: state,
            staleDate: startedAt.addingTimeInterval(Double(targetSeconds + 30))
        )
        do {
            activity = try Activity<RestTimerAttributes>.request(
                attributes: attributes,
                content: content,
                pushType: nil   // local-only updates; no APNs
            )
        } catch {
            // Simulator / denial / capability mismatch — silent fallback.
            activity = nil
        }
    }

    public func update(startedAt: Date, targetSeconds: Int) {
        // Cancel any in-flight debounce; schedule the latest mutation.
        debounceTask?.cancel()
        let snapshotActivity = activity
        let millis = debounceMillis
        debounceTask = Task { @MainActor [weak self] in
            try? await Task.sleep(for: .milliseconds(millis))
            guard !Task.isCancelled else { return }
            guard let _ = self, let activity = snapshotActivity else { return }
            let state = RestTimerAttributes.ContentState(
                startedAt: startedAt,
                targetSeconds: targetSeconds
            )
            let content = ActivityContent(
                state: state,
                staleDate: startedAt.addingTimeInterval(Double(targetSeconds + 30))
            )
            await activity.update(content)
        }
    }

    public func end() {
        debounceTask?.cancel()
        debounceTask = nil
        guard let activity else { return }
        let snapshot = activity
        let final = RestTimerAttributes.ContentState(
            startedAt: snapshot.contentState.startedAt,
            targetSeconds: 0   // signal "stopped"
        )
        Task { @MainActor in
            await snapshot.end(
                ActivityContent(state: final, staleDate: nil),
                dismissalPolicy: .immediate
            )
        }
        self.activity = nil
    }
}

/// No-op controller for unit tests + previews + simulator fallback.
@MainActor
public final class NoopActivityController: RestTimerActivityControlling {
    public init() {}
    public func start(startedAt: Date, targetSeconds: Int, exerciseName: String) {}
    public func update(startedAt: Date, targetSeconds: Int) {}
    public func end() {}
}
```

### Create — `FitbodWidgets/RestTimerLiveActivity.swift`

Per RESEARCH § Pattern 3 / Code Example 3 + UI-SPEC § Rest timer (Dynamic Island regions verbatim).

```
import ActivityKit
import WidgetKit
import SwiftUI

@available(iOS 16.1, *)
public struct RestTimerLiveActivity: Widget {
    public init() {}

    public var body: some WidgetConfiguration {
        ActivityConfiguration(for: RestTimerAttributes.self) { context in
            // Lock screen / banner view (iPhone < Pro, or all phones when
            // Dynamic Island can't render).
            RestTimerLockScreenView(
                state: context.state,
                exerciseName: context.attributes.exerciseName
            )
            .padding(16)                                                    // UI-SPEC § Spacing lg
            .activityBackgroundTint(Color(.secondarySystemGroupedBackground))
            .activitySystemActionForegroundColor(Color.accentColor)
        } dynamicIsland: { context in
            DynamicIsland {
                DynamicIslandExpandedRegion(.leading) {
                    Image(systemName: "timer")
                        .foregroundStyle(Color.accentColor)
                }
                DynamicIslandExpandedRegion(.trailing) {
                    Text(remainingText(state: context.state))
                        .font(.title2)
                        .fontWeight(.semibold)
                        .monospacedDigit()
                }
                DynamicIslandExpandedRegion(.center) {
                    Text(context.attributes.exerciseName)
                        .font(.caption)
                }
                DynamicIslandExpandedRegion(.bottom) {
                    ProgressView(
                        value: elapsed(state: context.state),
                        total: max(1, Double(context.state.targetSeconds))
                    )
                    .tint(Color.accentColor)
                }
            } compactLeading: {
                Image(systemName: "timer")
            } compactTrailing: {
                Text(remainingText(state: context.state))
                    .monospacedDigit()
            } minimal: {
                Image(systemName: "timer")
                    .foregroundStyle(Color.accentColor)
            }
            .widgetURL(URL(string: "fitbod://session/active"))
        }
    }

    private func elapsed(state: RestTimerAttributes.ContentState) -> Double {
        max(0, Date.now.timeIntervalSince(state.startedAt))
    }

    private func remainingText(state: RestTimerAttributes.ContentState) -> String {
        let remaining = max(0, Double(state.targetSeconds) - elapsed(state: state))
        let s = Int(remaining)
        return String(format: "%d:%02d", s / 60, s % 60)
    }
}
```

```
        // returns normally.
    }

    @Test("updateDebounceCoalesces — multiple updates fire one update")
    func updateDebounceCoalesces() async throws {
        // Verify the debounce window. The controller's debounce uses
        // an async sleep — wait long enough that the debounce fires.
        // The internal `activity` is nil on simulator, so the
        // `await activity.update(content)` line is short-circuited,

            controller.update(startedAt: Date.now, targetSeconds: delta)
        }
        try await Task.sleep(for: .milliseconds(100))
        // No crash = debounce machinery works.
    }
```

## Anti-Patterns to Avoid

- Do NOT call `Activity<RestTimerAttributes>.request(...)` outside a `do/catch` block. RESEARCH §6 Pitfall 3 — the call throws on simulators and pre-Pro iPhones; silent fallback is mandatory.
- Do NOT re-issue `Activity.request(...)` on every ±15s press. RESEARCH § Anti-Patterns: this creates a new Live Activity per press (Apple caps ~10 active activities per app). Use `activity.update(...)`.
- Do NOT skip the debounce on `update(...)`. RESEARCH §6 Pitfall 9 — Apple rate-limits Live Activity updates; bursts get dropped.
- Do NOT use a per-instance UUID for the notification identifier on the `RestTimerNotificationScheduler` (that's plan `02-01`'s concern, but also a soft constraint here: the Live Activity is a separate channel and doesn't share the identifier).
- Do NOT add the `RestTimerAttributes.swift` file to ONLY the main app target. ActivityKit requires the attribute type to be available in BOTH the requester (main app) AND the renderer (widget extension). Cross-target membership is non-negotiable.
- Do NOT put `RestTimerLiveActivity.swift` or `RestTimerLockScreenView.swift` in the main app target. Widget code lives in the widget extension target only.
- Do NOT bundle `WidgetKit.framework` or `ActivityKit.framework` (Embed mode). Both are system frameworks — link only.
- Do NOT use `App Groups` for cross-target state sharing. This phase does not need app groups; the widget receives state via the ActivityKit ContentState mechanism, not via a shared UserDefaults / file URL.
- Do NOT skip the `@available(iOS 16.1, *)` attribute on the widget types and the controller. The main app's deployment target is 18.0 so the guard is always-true at runtime, but the API surface itself requires the attribute for type checking.
- Do NOT call `liveActivity.end(...)` without `dismissalPolicy: .immediate`. Without it, the Dynamic Island lingers for several minutes after dismissal per Apple's default behavior — fine for some apps, wrong for a rest timer that should disappear the moment the next set begins.
