# Phase 2 — UI Design Contract

> Visual and interaction contract for Core Loop (Routines + Sessions). Generated by gsd-ui-researcher (autonomous mode), verified by gsd-ui-checker.

This phase delivers the minimum lovable product: a single-screen **RoutineBuilderView** with inline exercise search and per-exercise prescription editor, a **RoutinesListView** sectioned by folder, a **SessionLoggerView** with per-set inputs and inline "Previous" column, a **RestTimerLiveActivity** for the Dynamic Island and lock screen, and an **ExerciseHistoryView** with intent-split filter chips. The design stance remains the Phase 1 stance verbatim — **"comprehensive but uncluttered"**: favor information density that a serious lifter expects, lean on iOS-native chrome (system materials, SF Symbols, Dynamic Type, ActivityKit), and never introduce a custom design system where iOS already has one. Progressive disclosure is the tie-breaker when density and clutter conflict — tempo, partial reps, cluster sub-reps are opt-in per exercise (`RoutineExercise.tracksTempo: Bool` and friends), not always-on.

This UI-SPEC carries forward every token from Phase 1 (`01-UI-SPEC.md`) **verbatim**: the accent color, the 8-point spacing scale with 44pt HIG exception, the 4-role / 2-weight semantic typography, the 60/30/10 color split, and the terse-prescriptive copywriting voice. Anything below that doesn't restate a Phase 1 token is a Phase 2 addition needed for the seven new user-visible surfaces.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native SwiftUI — `components.json` / shadcn not applicable to iOS) |
| Preset | not applicable |
| Component library | SwiftUI built-ins (`Form`, `List`, `NavigationStack`, `Sheet`, `EditMode` / `.onMove`, `.searchable`, `Stepper`, `Picker`, `Slider`, `TextField`, `.confirmationDialog`, `.alert`, `Menu`) + Apple `ActivityKit` for the rest-timer Live Activity / Dynamic Island |
| Icon library | SF Symbols (Apple-bundled, weight matches typography) |
| Font | SF Pro — used via Dynamic Type semantic styles, never raw point sizes |

**Why "none":** Locked by `PROJECT.md` and `research/STACK.md` — zero third-party SPM / UI dependencies. The shadcn registry / preset model is React-only and not applicable to iOS native. New for Phase 2: **ActivityKit** is the only Apple framework added beyond Phase 1's surface. It is bundled with iOS and requires the `NSSupportsLiveActivities` Info.plist key + `com.apple.developer.activitykit` entitlement (no SPM).

## Spacing Scale

**Carries forward verbatim from Phase 1.** Multiples of 4, iOS 8-point grid.

| Token | Value | Phase 2 usage |
|-------|-------|---------------|
| xs | 4pt | Icon-to-label gaps (set type chip ↔ rep field), superset accent rail width, chip-padding inside a row, set-row chevron padding |
| sm | 8pt | Compact stack spacing — chip-to-chip in intent-filter row on history view, vertical gap between prescription chip and set rows, ±15s button padding |
| md | 12pt | Set-row vertical padding above/below content, routine-row vertical padding, prescription editor field gaps |
| lg | 16pt | Default `Form` / `List` horizontal inset, primary content margin from screen edge, gap between session header and first exercise card |
| xl | 24pt | Section spacing inside the routine builder between exercises (when not super-set-grouped), spacing between session-summary card and exercise list |
| 2xl | 32pt | Empty-state vertical padding above heading on empty Routines tab / empty ExerciseHistory |
| 3xl | 48pt | Top spacer above empty-state heading on a tall device |

**Exceptions (load-bearing for iOS HIG and Phase 2 specifics):**

- **44pt minimum touch target** for every tappable control. Specifically: each per-set checkmark, each RPE chip in the inline RPE row, each ±15s button on the rest timer, every drag handle in the routine builder (`line.3.horizontal` glyph with hit area padded to 44pt × 44pt), every intent-filter chip on the history view. Internal visual padding may be smaller (e.g. an RPE chip's label is 8pt-padded inside its capsule) but `.contentShape(Rectangle()).frame(minHeight: 44)` extends the hit area.
- **Set-row checkbox / completion glyph** is 32pt visually (matches system `Toggle`) but 44pt hit area.
- **Rest-timer overlay height when collapsed** is 64pt (a Dynamic Island-style pill that hovers above the tab bar). Expanded sheet detent is `.medium` (≈ half-screen).
- **Live Activity** layouts follow Apple's `ActivityKit` dimensions strictly — the compact / minimal / expanded Dynamic Island presentations have system-imposed bounds (compact ≤ 116pt wide trailing region, minimal = 22pt circle). No Phase 2 token overrides these.

## Typography

**Carries forward verbatim from Phase 1.** Dynamic Type semantic styles, never hard-coded pt sizes. Exactly **4 roles**, exactly **2 weights** (regular / semibold). No bold weight (`.heavy` / `.black`).

| Role | SwiftUI semantic style | Approx pt @ Large | Weight | Line height | Phase 2 usage |
|------|------------------------|-------------------|--------|-------------|---------------|
| Caption | `.caption` | 12 | regular (400) | 1.3 | "Previous" column data (`175 × 8 @ 8 (last Mon)`), set-type chip label, intent chip label on history rows, rep-range hint under prescription field, footer copy under "Why this rest?" disclosure, Live Activity supporting text |
| Body | `.body` | 17 | regular (400) | 1.4 | Set-row inputs (weight / reps / RPE values), exercise name in routine builder, exercise name in session logger, routine name in Routines tab, per-exercise notes inline, prescription editor field values, history row content |
| Headline | `.headline` | 17 | semibold (600) | 1.3 | Section headers in `List` (folder name in Routines tab, date header in history view), exercise card header in session logger, prescription editor section labels ("Sets / Reps", "Target RPE", "Rest"), navigation bar inline title (current routine name during build, "Workout" during session) |
| Title2 | `.title2` | 22 | semibold (600) | 1.2 | Rest timer countdown digits in the overlay, session-summary "elapsed" digits, empty-state heading on Routines / History, navigation bar large title ("Routines" tab root, "History" inside an exercise detail) |

No new role added in Phase 2. The rest-timer numeric countdown intentionally uses `.title2` (semibold) rather than introducing a fifth role — it reads as a strong number on the overlay without breaking the 4-role contract.

## Color

**Carries forward verbatim from Phase 1.** The 60/30/10 split is implemented via system materials with one designated accent. Light-mode hex values shown; dark-mode equivalents auto-applied by the same semantic asset references.

| Role | Light value | Dark value | Source | Phase 2 usage |
|------|-------------|------------|--------|---------------|
| Dominant (60%) | `#FFFFFF` | `#000000` | `Color(.systemBackground)` | Routine builder background, session logger background, Routines tab list background, history list background |
| Secondary (30%) | `#F2F2F7` | `#1C1C1E` | `Color(.systemGroupedBackground)` | Inside grouped `Form` / sectioned `List` (Routines tab, routine builder's prescription editor sheet), folder section background, history-row card background, Live Activity background (auto), rest-timer overlay surface |
| Accent (10%) | `#0E7C86` | `#3FBFC9` | `Asset: AccentColor` | (see "Accent reserved for" below — Phase 2 adds 5 specific surfaces to the Phase 1 list, never broadens the rule) |
| Destructive | `#FF3B30` | `#FF453A` | `Color(.systemRed)` | "Delete routine" confirmation button, "Discard workout" confirmation, swipe-to-delete action surfaces |
| Tertiary text | `#3C3C43` @ 60% opacity | `#EBEBF5` @ 60% opacity | `Color(.secondaryLabel)` | "Previous" column values (faint, behind/beside the actual input), inactive set-type chip text, prescription footer help, history row metadata (workout name + date stamp) |

**Accent asset reuse:** The same `AccentColor.colorset` populated in Phase 1 (already wired to the deep teal `#0E7C86` / `#3FBFC9`) propagates through every new control in Phase 2 — selected intent-filter chips, the superset accent rail, the active set-row completion glyph, the rest-timer progress arc, and the Live Activity progress mark all use it. **No new color asset is created in Phase 2.**

### Accent reserved for (Phase 2 surfaces; appended to the Phase 1 list)

The accent appears **only** at the following elements. Every other interactive control uses default system styling (`.secondaryLabel` inactive, `.label` active).

Phase 1 surfaces (still reserved):
1. Selected filter chip background fill (library facet picker) ✓
2. Selected `TabView` tab icon + label ✓
3. Primary action button in custom-exercise editor: "Save Custom Exercise" ✓
4. "Copy as Custom" secondary action in exercise detail view ✓
5. Active `Toggle` in Settings ✓
6. Active text cursor + selection in `.searchable` and form text inputs ✓
7. Stimulus-weight `Slider` track fill in custom-exercise muscle-mapping row ✓

Phase 2 additions:

8. **Selected intent filter chip** in the per-exercise history view (one chip from `All / Strength / Hypertrophy / Power / Endurance / Technique`); unselected chips use `Color(.systemGray5)`.
9. **Superset / giant-set accent rail** — a 4pt-wide vertical bar on the left edge of every exercise card that belongs to a superset group, in the accent color. Distinguishes paired/grouped exercises at a glance.
10. **Completed set-row checkmark glyph** (`checkmark.circle.fill` filled in accent) — the visual signal that this set is done. Pending sets show `circle` outlined in `.secondaryLabel`.
11. **Rest timer progress arc** in the overlay and in the Dynamic Island expanded presentation — a circular progress ring sweeping accent color around a `.title2` numeric countdown.
12. **Save button** ("Save Routine") in the routine builder's toolbar leading position (filled, accent background, white label) — analogous to Phase 1's "Save Custom Exercise" button.
13. **Primary "Start Workout" CTA** on the routine row long-press menu and on the routine detail view (filled, accent background, white label).
14. **"+ Folder" toolbar entry** in the Routines tab (text-only, accent foreground) — matches Phase 1's "Create Custom Exercise" text-button treatment.
15. **Intent chip on the routine builder exercise card** (e.g. "Strength" / "Hypertrophy") — capsule fill `Color.accentColor.opacity(0.15)`, label in accent — matches the Phase 1 "Custom" tag treatment.
16. **Per-exercise prescription "edit" disclosure chevron** when the editor row is expanded (system `DisclosureGroup` follows tint).
17. **Live Activity progress mark / Dynamic Island arc** — the rest-timer arc in the Dynamic Island expanded presentation (and the small leading-icon dot in the compact presentation).

**Explicitly NOT accent:**
- Drag handle (`line.3.horizontal`) — uses `.secondaryLabel`, never accent. It's a grip, not an action.
- Pending (incomplete) set-row checkmark — `.secondaryLabel`.
- The `±15s` rest-timer buttons — system `BorderedButtonStyle` on `.label`, never accent. Mutating rest is not a "primary action."
- "Skip rest" text button on the rest timer overlay — `.secondaryLabel` on system fill, never accent. Stopping rest early is incidental, not aspirational.
- The "Previous" column data — `.tertiaryLabel`, never accent. It's a reference, not a CTA.
- The set-type chip ("warmup" / "drop" / "failure" / "rest-pause" badges) — system-colored per Hevy's convention: warmup = `Color(.systemBlue)` capsule, drop = `Color(.systemOrange)`, failure = `Color(.systemRed)`, rest-pause = `Color(.systemPurple)`. Working sets render no chip. **These are semantic system colors, not "accent."** They never broaden the accent reserved-for list.
- Section header text in the Routines tab folder list — `.headline` on `.label`, never accent.
- Intent badge inline on a history row — uses a muted neutral fill (`Color(.systemGray6)` background, `.label` foreground) when listed alongside actual data. Only the **selected filter chip** above the list uses accent; the inline intent badges below are quiet.

### Interaction patterns

**Routines tab:**
- Routines tab tapped while already active → clear that tab's `NavigationPath` (same Phase 1 re-tap pop-to-root pattern).
- Long-press a routine row → context menu: "Start Workout" (accent), "Duplicate", "Move…", "Edit", "Delete".
- Swipe leading on a routine row → "Start Workout" (accent fill, white label, glyph `play.fill`).
- Swipe trailing on a routine row → "Delete" (destructive); secondary swipe action "Duplicate".
- Long-press a folder header → "Rename" / "Move Up" / "Move Down" / "Delete" (only enabled when empty).
- `+` toolbar → `Menu` with "New Routine" → push `RoutineBuilderView(draft: RoutineDraft())` and "New Folder" → present `NewFolderSheet` as `.sheet`.
- Tap a routine row → push `RoutineBuilderView(routine:)` in edit mode (NOT a sheet — full-screen edit so drag-reorder feels native).

**Routine builder:**
- Drag-handle reorder uses SwiftUI `EditMode` + `.onMove` — but the `EditMode` is implicit (always-on visible drag handles per CONTEXT.md, not toggled by an "Edit" button). The handle glyph is `line.3.horizontal` in `.secondaryLabel` color.
- Inline exercise search at the bottom of the exercise list — `.searchable(text:)` on the inner list with results surfaced as a `LazyVStack` *inside* the same `ScrollView` (not a sheet). Tap a result → append a new `RoutineExerciseDraft` to the list with prescription defaults.
- Prescription defaults heuristic (per CONTEXT.md Area 1): when an exercise is appended, intent defaults from the exercise's `mechanic` (`compound` → `strength` for barbell/dumbbell compounds, `hypertrophy` otherwise) and rest defaults from mechanic (compound = 180s, isolation = 90s).
- Tapping an exercise row toggles the inline `PrescriptionEditorRow` expansion (SwiftUI `DisclosureGroup`).
- Tap-and-hold on an exercise row → long-press menu with superset / duplicate / remove actions.
- Superset assignment: choose an existing group OR "New Superset" → creates a `SupersetGroup`, sets `RoutineExercise.supersetGroupID`, and the visual rendering updates immediately (accent rail appears on left edge of all grouped cards).
- "Cancel" with dirty form → `.confirmationDialog` with discard / keep-editing options.
- "Save" disabled when `RoutineDraft.isValid == false` (no name OR no exercises).

**Session logger:**
- Header chip layout is horizontal: `[clock 32:14] [3 of 6] [pencil Notes]`.
- Per-exercise card layout: exercise name `.headline`, optional pinned-note line `.caption` on yellow capsule, set-row column header (only first time per session), then list of `SetRow`s, then trailing "+ Add Set" text button (accent).
- Per-set row layout (horizontal): `[set# 1] [prev 175×8@8] [weight 180] [reps 8] [rpe chips] [type chip] [✓]`. On narrower screens (smaller iPhones / split-screen iPad), the prev column reflows to a second line under the set#.
- Inline RPE chip row: 5 chips (6/7/8/9/10) horizontal scrollable. Tap to set integer RPE. Long-press any chip → `DecimalRPEPickerSheet` opens with wheel picker for decimal values (centered on the long-pressed integer).
- Set type chip tap cycles `working → warmup → drop → failure → restPause → working`. Long-press → menu for direct choice.
- Tapping an empty weight cell prefills the matching-intent previous weight (from `PreviousColumn` query) — user types to override. Same for reps.
- Tapping the trailing completion checkmark on a set row commits the set (writes `SetEntry.completedAt = Date.now`, validates required fields `weight > 0 || isWarmup, reps > 0`) AND triggers `RestTimerEngine.start(seconds: prescribedRestForCurrentExercise)`.
- The completion checkmark turns from `circle` (`.secondaryLabel`) to `checkmark.circle.fill` (accent) on commit.
- When the user taps the *next* set's weight or reps cell (focus transition to a still-incomplete set), `RestTimerEngine.stop()` is called and the cancelled `UNUserNotification` is removed. This is the "auto-stop on next set entry" pattern.
- "+ Add Set" appends a new `SetEntry` with `orderIndex = current.count` and intent-snapshotted defaults from the exercise.
- Long-press an exercise card header → "Swap Exercise…" / "Edit Pinned Note" / "Remove from Session".
- Bottom "Add Exercise" button → opens `ExerciseLibraryView` in embedded mode; the chosen exercise is appended to the session (not the routine) with intent-defaulted prescription fields.
- "Finish" button → `.confirmationDialog` with summary; on confirm writes `Session.completedAt = Date.now` and pops back to Routines tab.
- "Discard" button (visible only when no sets have been logged yet) → `.alert` confirming discard; on confirm deletes the active `Session` (and its empty `SessionExercise` rows by cascade) and pops back.

**Rest timer:**
- Auto-starts on set completion. Mounts the `RestTimerOverlay` at the top of the screen (below the navigation bar, above the exercise list). Renders as a 64pt collapsed pill in the Phase 2 contract — Apple Music's now-playing pill is the visual reference.
- Tap the collapsed pill → expand to `.medium` detent sheet with `−15s` / `+15s` / `Skip` controls and an exercise-name subtitle.
- Adjust ±15s → `RestTimerEngine.targetSeconds += ±15` (does NOT move `startedAt`) — and reschedules the `UNUserNotification` for `startedAt + targetSeconds`.
- Skip rest → `RestTimerEngine.stop()`, cancels the notification, dismisses the overlay.
- When the timer reaches 0 — in-app: the overlay flashes the accent ring once, plays a system success haptic (`UINotificationFeedbackGenerator(.success)`), and stays mounted with "Rest complete" copy until the user taps it or starts the next set. Lock-screen: the scheduled `UNUserNotification` fires with the verbatim copy above ("Rest complete" / "{exercise name} — next set ready.").
- Live Activity: starts simultaneously with the in-app overlay if `ActivityKit` is supported and `NSSupportsLiveActivities` is `YES`. Updates every 1s using `Activity.update(using:)` with the new `ContentState`. Auto-ends when the timer is stopped or the session is finished.
- Dynamic Island compact presentation when the user is on the home screen: small `timer` icon leading + countdown trailing. Long-press → expanded layout with progress arc + exercise name.

**Exercise history:**
- Entry: from `ExerciseDetailView` (Phase 1) — add a "History" section at the bottom with a "View All History" `NavigationLink` that pushes `ExerciseHistoryView` onto the Library tab's `NavigationStack`.
- Intent filter chip row pins to the top of the list (`safeAreaInset(edge: .top)`).
- Default selection: "All" (showing every logged set across all intents).
- Tap a chip → updates the `@Query` predicate (or the filtered subset within the cached query — implementation detail; the UI contract is "rows visibly update").
- Date sections are `.headline`-styled headers (e.g. "Mon, May 11").
- Tap a history row → no navigation in Phase 2 (deferred to Phase 6 session detail view). The row is read-only.

## Accessibility Contract

The user is the only user and is sighted, but iOS HIG accessibility behaviors come "for free" with native SwiftUI and breaking them would be self-sabotage on a gym floor where one-handed / Dynamic Type / VoiceOver / reduce-motion scenarios are real. The Phase 1 accessibility contract applies; Phase 2 adds the following surfaces:

| Surface | Contract |
|---------|----------|
| Drag handle on routine-builder rows | `accessibilityLabel` = "Reorder {exercise name}". `accessibilityHint` = "Double-tap and drag to reorder". The drag interaction itself uses SwiftUI's native `.onMove` which VoiceOver handles with Rotor / "Move Up / Move Down" actions automatically. |
| Set-row checkmark | `accessibilityLabel` = "Mark set {N} complete" / "Set {N} complete". `accessibilityTraits` = `.button` + `.isSelected` when complete. |
| Intent filter chip | `accessibilityLabel` = "{intent name} filter, {currently selected / unselected}". `accessibilityTraits` = `.button` + `.isSelected` when selected. |
| RPE chip row | Each chip has `accessibilityLabel` = "RPE {N}". `accessibilityHint` (only on first chip) = "Double-tap and hold for decimal RPE". |
| Rest timer overlay (collapsed pill) | `accessibilityLabel` = "Rest timer: {remaining time}". `accessibilityHint` = "Tap to expand controls". `accessibilityValue` = "{remaining seconds}". `accessibilityAdjustableAction` triggers ±15s. |
| Rest timer overlay (expanded sheet) | `±15s` buttons have `accessibilityLabel` = "Add 15 seconds" / "Subtract 15 seconds". "Skip" has `accessibilityLabel` = "Skip remaining rest". |
| Set-type chip | `accessibilityLabel` = "{set type name}". `accessibilityHint` = "Tap to cycle set type. Tap and hold for menu". |
| Pinned note inline | `accessibilityLabel` = "Pinned note: {note text}". `accessibilityHint` = "Tap to edit". |
| Live Activity / Dynamic Island | All `ActivityKit` content uses `accessibilityLabel` on each element (countdown, exercise name, progress arc) — Apple's `WidgetKit` accessibility patterns apply. |
| Dynamic Type | All copy uses semantic styles (Phase 1 token); set-row layouts wrap on accessibility-large sizes — weight/reps/RPE cells flow to a second line rather than truncating. |
| Reduced motion | The rest-timer progress arc animation respects `@Environment(\.accessibilityReduceMotion)` — when on, the arc snaps to discrete states instead of sweeping. Dynamic Island animations are Apple-managed and respect the same setting. |
| Color | Accent (`#0E7C86` / `#3FBFC9`) clears WCAG AA on `systemBackground` and `systemGroupedBackground` (verified in Phase 1). The set-type chip colors (blue/orange/red/purple) are system semantic colors and meet contrast requirements by Apple's guarantee. |
| Haptics | The set-row commit fires `UINotificationFeedbackGenerator(.success)`. The rest-timer reaches-zero event fires `UINotificationFeedbackGenerator(.success)`. Both respect Settings → Accessibility → System Haptics (system-managed). |
